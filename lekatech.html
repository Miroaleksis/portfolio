Assignment

Assignment for our entire team was to come up with two hammer concepts and two hammer UI concepts. The hammer-team and the UI-team worked mostly separately but with constant check ups and critiques with each other. Names of the finnished hammer concepts were Agitator M3 and GranitE3.

Basic info about the breaker hammers

Breaker hammers are used in excavators. Earlier the only breaker hammers in the market were hydraulic hammers so electric hammers were a new invention. With both hammers the hammering starts when user presses a pedal with their foot. The difference is that electric hammer can be controlled via touchscreen device (device where we made the UI for) on the excavator so that the impact energy and the frequency of hammering can be changed. This makes hammering throught different kinds of materials easier.

UI Concepts

One UI concept was not specifically linked to one hammer concept but rather we made one "conservative" UI close to customers pre-existing drafts and the other was more futuristic with more liberties in execution. We ended up calling the conservative UI Enhanced and the futuristic one Innovative. When prototyping these concepts, I was responsible for the Innovative UI so I only have deeper dive to that concept in this page.

Process

We started by studying all the key functions of the hammers and the environment UI was used on (excavator cabin). Not much user research was possible since the electric hammers didn’t exist on the market yet and therefore there were no pre-existing users and the project was confidential so we couldn’t share any information about it. We did manage to get some information from construction workers about hammering related things.

Findings

Excavator drivers often wear gloves, sometimes work in dark and the hammering is a pretty straight-forward routine procedure. That meant that the UI needed to...

  • be visually clear and perceicable by one glance, easy to bring in to use
  • require mimimal clicks and be usable with gloves on
  • have dim colours so light of the screen won't dazzle the user

Prototyping

The prototyping process was iterative, we made some wireframes first and then showed them to the customer, re-developed the necessary parts and then showed an updated version until the final UI was ready.

Innovative UI

Operating screen

Since the most crucial function of the UI was the hammering, we decided the operation screen was to be the first thing you saw when opening the device so you could immediately start to work.

Changing program

Having different programs is the key function separating electric breaker hammer from hydraulic hammer (competitor). Programs help hammering different kind of materials with most efficient hammering frequency and impact energy. Programs are divided into modes and cycles. Modes are programs with only one type of frequency and impact energy, and cycles are combination of many modes.

Physical buttons

Physical buttons were part of the device the UI was made for so we decided to use them as "backup" buttons in case user was using gloves and couldn't touch the touchscreen. We put all the navigation links there plus shortcut to three most commonly used modes.
