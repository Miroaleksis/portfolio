Assignment

Assignment from to the our entire team was to come up with two hammer concepts and two hammer UI concepts. The hammer-team and the UI-team worked mostly separately but with constant check ups and critiques with each other. Names of the finnished hammer concepts were Agitator M3 and GranitE3.

Basic info about the breaker hammers

The remote control of the hammer happens from excavator machine cabin where the touchscreen-device is located. The hammering starts when user presses a pedal in the cabin with their foot. So there is no "Start hammering" type button in the UI. Note: UI also includes physical buttons.

UI Concepts

One UI concept was not to be specifically linked to one hammer concept but rather we made one "conservative" UI close to customers pre-existing drafts and vision and the other was more futuristic with more liberties in execution. We ended up calling the conservative UI Enhanced and the futuristic one Innovative. When prototyping these concepts, I was responsible for the Innovative UI. This is why I only have deeper dive to the Innovative in this page.

Enhanced

  • Flow of the UI kept close to customers vision
  • Safe option for acute need
  • Updated visuals

Innovative

  • Entire structure and visuals updated
  • Flow of the UI imitating other modern UI:s
  • Extra functions such as user accounts and different rights on different users.

Innovative UI

Opening/operating screen

Since the most crucial function of the UI was the hammering, we decided the operation screen was to be the first thing you saw when opening the device so you could immediately start to work.

Changing program

Having different programs is the key function separating electric breaker hammer from hydraulic hammer (competitor). Programs help hammering different kind of materials with most efficient hammering frequency and impact energy. Programs are divided into modes and cycles. Modes are programs with only one type of frequency and impact energy, and cycles are combination of many modes.

Note! Icons used in the modes are placeholders and were to be replaced in further development of the UI.

Physical buttons

Physical buttons were part of the device the UI was made for so we decided to use them as "backup" buttons in case user was using gloves and couldn't touch the touchscreen. We put all the navigation links there plus shortcut to three most commonly used modes.
